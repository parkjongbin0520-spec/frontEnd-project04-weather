# 👕 입을만해 (Ib-eul-man-hae)

> \*\*"숫자만으로는 부족한 날씨 정보, 이제 눈으로 직접 확인하고 외출하세요."

---

## 🎯 1. 기획 배경 및 해결 과제

- **현재 상황 및 문제점:** 외출 전 날씨에 맞는 옷을 고르는 데 5~10분의 시간을 허비하며, 기온에 맞지 않는 옷을 입어 덥거나 추워 고생하는 경우가 빈번함. 또한, 옷을 잘 못 입는 사람(패알못)은 추천된 스타일 중 자신에게 없는 아이템을 어떻게 대체해야 할지 막막해함.
- **우리가 제공할 해결책:** 위치 기반 기온 데이터를 활용해 '체감 온도별 최적의 코디'를 즉각 제안함. 사용자의 옷장에 해당 아이템이 없을 경우를 대비해 대체 아이템 가이드를 제공하며, 필요 시 바로 구매할 수 있는 쇼핑 링크를 연결함.

## 👤 2. 타겟 유저 페르소나

- **인구통계학적 특성:** 20 30대 사회초년생 및 대학생, 아침 출근/등교 시간이 촉박한 직장인

- **불편함:** 오늘 15도라는데 셔츠만 입어도 되나?" 하는 온도 감각의 부재

- **기대하는 것:** 고민 없이 시키는 대로만 입어도 "오늘 옷 깔끔하네" 소리를 듣는 것, 그리고 외출 준비 시간의 단축.

## ⭐ 3. 핵심 기능

> 프론트엔드 환경에서 3주 내에 구현할 수 있는 핵심적인 기능 딱 3가지를 정의합니다.

1. [기온별 OOTD 추천]: 현재 위치의 기상 API를 연동하여 기온에 맞는 '표준 코디' 이미지를 카드 형태로 보여줍니다. (예: 10~12도 → 자켓, 가셔츠, 면바지)
2. [대체 아이템 '스위칭' 기능]: 추천된 코디 중 집에 없는 아이템이 있다면 '대체하기' 버튼을 눌러 비슷한 느낌의 다른 아이템(가디건 대신 얇은 경량 패딩 등)을 추천받습니다.
3. [원클릭 커머스 연동]: 추천 코디와 유사한 상품을 무신사나 지그재그 등 외부 쇼핑몰 검색 결과로 바로 연결해 주는 '내 옷장 채우기' 버튼을 제공합니다.

## 🗺️ 4. 화면 설계 및 유저 플로우

- 사용자가 우리 웹사이트에 접속하여 목표를 달성하기까지의 클릭 흐름을 단계별로 작성합니다.

  1. `메인 페이지` 접속 : 현재 위치의 온도와 날씨 아이콘, 그리고 대표 추천 코디 1종 노출.

  2. `상세 가이드` 클릭: 상의/하의/외투별 구체적인 소재(린넨, 기모 등)와 추천 아이템 리스

트 확인.

3. `대체 아이템` 확인: 본인에게 없는 옷을 클릭하여 대체 가능한 스타일 조합 확인.

4. `구매 페이지` 이동: 마음에 드는 코디 아이템의 구매 버튼 클릭 시 외부 쇼핑몰 페이지로 전

   환
